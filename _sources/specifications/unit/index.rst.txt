Unit Test Specifications
=========================

This section contains unit test specifications for individual software components.

Overview
--------

Unit tests verify the correct behavior of individual functions and modules in isolation.

Test Environment
----------------

* **Framework**: Unity Test Framework
* **Coverage Tool**: gcov/lcov
* **Target**: Host PC (x86) and Traveo II MCU
* **Mock Framework**: CMock for hardware abstraction

Communication Module Tests
---------------------------

.. spec:: CAN Initialization Test
   :id: SPEC_UNIT_001
   :status: implemented
   :links: REQ_FUNC_001
   :verification: test
   :component: Communication
   :tags: unit, can
   
   **Objective**: Verify CAN controller initialization
   
   **Preconditions**:
   
   * CAN hardware is available or mocked
   * System clock is configured
   
   **Test Steps**:
   
   1. Call CAN_Init() with valid configuration
   2. Verify return value is E_OK
   3. Read back CAN controller registers
   4. Verify bitrate is set to 500 kbit/s
   5. Verify CAN controller is in normal mode
   
   **Expected Results**:
   
   * Initialization succeeds
   * Bitrate configured correctly
   * Controller enters normal operational mode
   
   **Test Code**:
   
   .. code-block:: c
   
      void test_CAN_Init_Success(void)
      {
          Can_ConfigType config = {
              .baudrate = 500000,
              .mode = CAN_MODE_NORMAL
          };
          
          Std_ReturnType result = CAN_Init(&config);
          
          TEST_ASSERT_EQUAL(E_OK, result);
          TEST_ASSERT_EQUAL(500000, CAN_GetBaudrate());
          TEST_ASSERT_EQUAL(CAN_MODE_NORMAL, CAN_GetMode());
      }

Sensor Module Tests

.. spec:: ADC Reading Test
   :id: SPEC_UNIT_010
   :status: implemented
   :links: REQ_FUNC_010
   :verification: test
   :component: Sensors
   :tags: unit, adc
   
   **Objective**: Verify analog sensor reading accuracy
   
   **Test Steps**:
   
   1. Configure ADC channel
   2. Set simulation input voltage to 2.5V
   3. Trigger ADC conversion
   4. Read converted value
   5. Calculate voltage from digital value
   6. Verify accuracy within ±1%
   
   **Expected Results**:
   
   * Conversion completes successfully
   * Calculated voltage: 2.5V ± 0.025V
   * Conversion time < 100µs

.. spec:: Temperature Sensor Plausibility Test
   :id: SPEC_UNIT_011
   :status: implemented
   :links: REQ_FUNC_011
   :verification: test
   :component: Sensors
   :tags: unit, sensors
   
   **Objective**: Verify temperature sensor plausibility checking
   
   **Test Cases**:
   
   1. **Valid temperature**: Input 25°C → Output 25°C, no error
   2. **Low out-of-range**: Input -50°C → Error flag set
   3. **High out-of-range**: Input 150°C → Error flag set
   4. **Sensor short circuit**: Input 0V → Fault detected
   5. **Sensor open circuit**: Input 5V → Fault detected
